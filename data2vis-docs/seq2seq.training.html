seq2seq.training.utils.create_input_fn(pipeline, batch_size, bucket_boundaries=None, allow_smaller_final_batch=False, scope=None)[source]¶
Creates an input function that can be used with tf.learn estimators.
Note that you must pass “factory funcitons” for both the data provider and featurizer to ensure that everything will be created in the same graph.
Parameters:
  • pipeline – An instance of seq2seq.data.InputPipeline.
  • batch_size – Create batches of this size. A queue to hold a reasonable number of batches in memory is created.
  • bucket_boundaries – int list, increasing non-negative numbers. If None, no bucket is performed.
Returns:

An input function that returns (feature_batch, labels_batch) tuples when called.
seq2seq.training.utils.create_learning_rate_decay_fn(decay_type, decay_steps, decay_rate, start_decay_at=0, stop_decay_at=1000000000.0, min_learning_rate=None, staircase=False)[source]¶

Creates a function that decays the learning rate.

Parameters:
  • decay_steps – How often to apply decay.
  • decay_rate – A Python number. The decay rate.
  • start_decay_at – Don’t decay before this step
  • stop_decay_at – Don’t decay after this step
  • min_learning_rate – Don’t decay below this number
  • decay_type – A decay function name defined in tf.train
  • staircase – Whether to apply decay in a discrete staircase, as opposed to continuous, fashion.
Returns:

A function that takes (learning_rate, global_step) as inputs and returns the learning rate for the given step. Returns None if decay_type is empty or None.
